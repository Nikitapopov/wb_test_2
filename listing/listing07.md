Что выведет программа? Объяснить вывод программы.

```go
package main

import (
	"fmt"
	"math/rand"
	"time"
)

func asChan(vs ...int) <-chan int {
	c := make(chan int)

	go func() {
		for _, v := range vs {
			c <- v
			time.Sleep(time.Duration(rand.Intn(1000)) * time.Millisecond)
		}

		close(c)
	}()
	return c
}

func merge(a, b <-chan int) <-chan int {
	c := make(chan int)
	go func() {
		for {
			select {
			case v := <-a:
				c <- v
			case v := <-b:
				c <- v
			}
		}
	}()
	return c
}

func main() {

	a := asChan(1, 3, 5, 7)
	b := asChan(2, 4 ,6, 8)
	c := merge(a, b )
	for v := range c {
		fmt.Println(v)
	}
}
```

Ответ:
```
Вывод: числа от 1 до 8 в случайном порядке, причем как четные, так и нечетные упорядочены по возрастанию. Далее программа зависает из-за дедлока.

Создаются два канала-читателя a и b. Функция asChan принимает последовательность чисел, которые последовательно в горутине передаются в небуферизированный канал через случайный промежуток времени до 1 секунды.

Далее в функции merge данные из двух каналов-читателей передаются в один канал в горутине с помощью механизма селект. Селект случайным образом читает один из готовых к чтению каналов и пишет в канал c, который и возвращается из функции merge.

Далее выполняется итерацию чтения из канала c с последующим выводом значений. Но так как канал c, в отличие от a и b, не закрывается после окончания записи, образуется дедлок в строке 46.

Как исправить: проверять каналы в строках 31 и 33 на закрытие, и в положительном исходе обнулять их. В цикле for проверять на необнуленность каналов, и если оба канала равны nil, то закрывать канал c и выходить из цикла.
Реализация:

func merge(a, b <-chan int) <-chan int {
	c := make(chan int)
	go func() {
		for a != nil || b != nil {
			select {
			case v, ok := <-a:
				if !ok {
					a = nil
					continue
				}
				c <- v
			case v, ok := <-b:
				if !ok {
					b = nil
					continue
				}
				c <- v
			}
		}
		close(c)
	}()
	return c
}
```
