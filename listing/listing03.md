Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
Вывод: nil, false
Интерфейс представляет из себя два указателя. Первый ссылается на структуру itable, в которой содержится метадата об интерфейсе, информация, какие методы должны быть реализованы. Второй содержит информацию о статическом типе фактических данных. Соответственно, для пустого интерфейса itable не задан.

В строчке 12 у переменной err задается конкретный тип *os.PathError, который уже указывает на itable, несмотря на то, что значение приравнивается к nil.

Первый вывод выводит значение err = nil.
Второй вывод сравнивает err и nil. Вывод будет false, так как err будет равна nil только, если и значение, и тип будут равны nil. В нашем случае тип не равен nil.

Как исправить: задать тип у переменной err, как error. Или возвращать не переменную err типа *os.PathError, а nil.
```
